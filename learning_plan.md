# AI Agent Memory 方向学习规划

> 制定时间：2026-04-01
> 适用周期：近期（0-2个月）学习冲刺期，中期（3-6个月）分叉期，远期（6-12个月）发展期

---

## 近期（0-2个月）：深度消化综述 + 持续展示

### 第1周：建立学习基础设施

**Agent/Memory 方向**
- [ ] 通读第一篇综述，不做深读，标记：
  - 完全不熟悉的概念（需补课）
  - 和你工程背景相关的点（你的优势切入点）
  - 你认为可以延伸/质疑的地方
- [ ] 创建学习记录文档，格式参考：
  ```
  ## 综述名
  ### 核心脉络（用自己的话概括）
  ### 不懂的概念清单
  ### 工程视角的思考
  ### 可延伸的问题
  ```

**英语**
- [ ] 每天朗读综述的 Introduction 20分钟（熟悉学术表达）
- [ ] 整理20个高频学术短语到自己的词库

**LLM 基础**
- [ ] 列出综述中提到的所有模型/技术，分类：熟悉/听过但不熟/完全陌生

---

### 第2-3周：深度精读 + 第一次反馈

**Agent/Memory 方向**
- [ ] 精读第一篇综述的核心章节（与你的工程背景最相关的部分）
- [ ] 针对每个关键机制，问自己：如果用工程系统实现，瓶颈在哪？
- [ ] 输出第一篇学习笔记（1500字），结构：
  1. 综述核心贡献一句话总结
  2. 技术脉络图（手绘或 mermaid）
  3. 一个工程视角的观察（如"Memory检索的延迟问题"）
  4. 一个待验证的问题或想法

**交流**
- [ ] 给他人发交流，附上你的笔记，请教不懂的具体问题
- [ ] 问题要体现你读了且想了，比如："关于文中提到的XX机制，我注意到工程实现上可能有XX问题，不知道实际研究中是如何权衡的？"

**英语**
- [ ] 背诵 Introduction 中5个经典长句，用于自己的学术写作
- [ ] 能够用英文向介绍你目前的工作内容（1分钟版本）

**LLM 基础**
- [ ] 针对综述中提到的3个最陌生的基础概念，各找1篇 tutorial 补完
- [ ] 输出简短总结，证明你懂了

---

### 第4-5周：第二篇综述 + 对比思考

**Agent/Memory 方向**
- [ ] 通读第二篇综述
- [ ] 输出对比分析：两篇综述的视角差异、互补点、矛盾点
- [ ] 输出第二篇学习笔记，结构：
  1. 与第一篇的对比表
  2. 该综述的独特贡献
  3. 一个结合两篇的延伸思考

**与导师互动**
- [ ] 第二次联系导师，分享对比思考，询问："这两篇综述在XX问题上的视角似乎有差异，您更认同哪种思路？"

**LLM 基础**
- [ ] 如果综述涉及实验，尝试理解实验设计：用了什么数据集、评价指标、baseline
- [ ] 思考：如果用更真实的工程场景评估，这些实验设计有什么问题？

---

### 第6-8周：输出个人思考 + 阶段收尾

**Agent/Memory 方向**
- [ ] 输出第三篇笔记：《工程视角下的Agent Memory系统设计考量》
  - 结合你的微服务/部署经验
  - 提出一个你认为被现有研究忽视的问题
- [ ] 尝试复现综述中一个简单的实验或机制（哪怕是 toy example）

**交流互动**
- [ ] 第三次联系，分享你的系统思考或复现结果
- [ ] 明确表达深造意愿，询问是否有进一步的准备建议

**英语**
- [ ] 用英文讲解你的工程背景和为什么想转研究方向
- [ ] 如果可能，约一次和导师的英文视频沟通

## 中期（3-6个月）：分叉执行

### 路径A：学术深造

**第9-12周：入学前准备**

| 周次 | 具体任务 | 产出物 |
|------|---------|--------|
| 9 | 自己确认研究方向，预读实验室3篇代表作 | 3篇论文笔记 |
| 10 | 搭建实验室代码环境，跑通基础demo | 可运行的环境+记录 |
| 11 | 学习实验室技术栈（工具/框架/数据） | 技术笔记 |
| 12 | 讨论入学后的第一个小课题 | 初步研究计划 |

**第13-24周：研究起步**
- 每两周读2-3篇领域论文，维持文献阅读习惯
- 开始第一个探索性实验（哪怕失败也要有记录）
- 补数学基础：线性代数、优化、概率论（按需，用到什么补什么）

### 路径B：求学不顺 → 求职

**第9-12周：技能补强**

| 方向 | 具体任务 | 产出物 |

- 每周2-3个岗位投递，优先 Agent/Memory 相关方向

- 准备系统设计：LLM服务架构、RAG系统设计、Agent编排系统

## 关键取舍原则

1. **不追求完美**：笔记有观点即可，不必等"学完"再写
2. **优先展示价值**：每篇内容都要体现你的工程背景优势
3. **英语实用主义**：现在只为申博服务，不系统学语法
