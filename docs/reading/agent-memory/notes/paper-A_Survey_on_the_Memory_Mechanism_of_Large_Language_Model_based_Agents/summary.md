# A Survey on the Memory Mechanism of Large Language Model based Agents — 阅读笔记

> 阅读时间：2026-04-10
> 关联方向：Agent Memory

---

## 摘要

本文是一篇关于大语言模型（LLM）智能体记忆机制的综述，系统梳理了 Agent Memory 的定义、来源、形式、操作、评估方法及应用场景。作者从认知心理学、自我进化和智能体应用三个动机出发，提出了一个统一的记忆演化框架：记忆写入（Writing）→ 记忆管理（Management）→ 记忆读取（Reading）。

---

## 我的理解

这篇综述的核心是建立一个**形式化的记忆机制分析框架**。作者把智能体与环境的交互抽象成三个连续阶段：感知（从环境获取信息）→ 处理（将信息转化为可用记忆）→ 行动（基于记忆做出决策）。而记忆模块正是通过 **写入-管理-读取** 三个操作来支撑这一循环的。

关键的理解点是：
- **狭义记忆** = 同一个 trial 的历史信息（类似当前会话上下文）
- **广义记忆** = 跨 trial 信息 + 外部知识 + 负反馈经验 + 用户偏好等
- 记忆不是简单的历史记录拼接，而是一个**动态的、可演化的系统**

## 核心定义

### 1. 智能体与环境交互

智能体完成任务的完整流程分为三个阶段：
1. **从环境中获取信息**（observation）
2. **对信息进行处理**，使其可用
3. **基于信息采取行动**（action）

记忆模块通过 **记忆写入、记忆管理、记忆读取** 实现第二阶段。

### 2. 任务与环境

- **任务（Task）**：智能体的最终目标
- **环境（Environment）**：智能体为完成任务而交互的对象，任何能影响智能体决策的情境因素

### 3. Trial 与 Step

- **Trial**：完整的智能体-环境交互过程
- **Step**：智能体-环境交互的单个回合

### 4. Agent Memory 的两种定义

| 定义 | 范围 |
|------|------|
| **狭义（Narrow）** | 同一个 trial 内的历史信息 |
| **广义（Broad）** | 跨不同 trials 的信息 + 外部知识 + 负反馈经验 + 用户偏好/特点/历史行为/知识库 |

## 统一演化函数

作者提出的统一记忆演化公式：

$$
m_{t}^{k} = W(\{a_{t}^{k}, o_t^k\})
$$

$$
M^k_t = P(M^k_{t-1}, m_t^k)
$$

$$
\hat{M}_t^k = R(M_t^k, c_{t+1}^k)
$$

$$
a_{t+1}^k = R\left(P\left(M^k_{t-1}, W(\{a_t^k, o_t^k\})\right), c_{t+1}^k\right)
$$

其中：
- $W$：**记忆写入（Writing）**，将原始交互 $\{a_t^k, o_t^k\}$ 映射为记忆内容 $m_t^k$
- $P$：**记忆管理（Management）**，对历史记忆 $M_{t-1}^k$ 和新记忆 $m_t^k$ 进行迭代处理
- $R$：**记忆读取（Reading）**，根据当前上下文 $c_{t+1}^k$ 从记忆中检索相关信息
- 输出：支撑下一步动作 $a_{t+1}^k$

### 记忆管理的三个子操作

1. **提炼高级知识**：增加泛化能力
2. **融合相似知识**：减少冗余
3. **Weed memory**：删除无用/不相关的记忆信息，提升准确性

## 为什么需要记忆机制

### 1. 认知心理学（Cognitive Psychology）

认知心理学研究注意力、语言使用、记忆、感知、问题解决、创造力和推理。记忆是认知中最重要的部分，是提炼高级知识、学习等的基础。设计智能体的目的是替代人类部分工作，因此需要遵从人类的工作机制——记忆对人类至关重要，所以为智能体设计记忆机制非常重要。

### 2. 自我进化（Self-evolution）

完成不同任务需要在动态环境中进行自我进化，而记忆对于自我进化至关重要。记忆需要支撑：
- **经验积累**：失败的经验、成功的经验
- **环境探索**：从不同行动中得到反馈并学习，通过历史信息决定如何探索
- **知识抽象**：从原始观测抽象出高层信息，增强对未知环境的适应力和泛化能力

### 3. 智能体应用（Agent Application）

使用智能体时，对话需要有历史信息，这是进行下一轮回复的必要条件。

## 如何实现记忆机制

### 记忆来源（Memory Sources）

记忆内容来自三类来源：

| 来源 | 性质 | 作用 |
|------|------|------|
| **单次实验信息（Inside-trial）** | 动态 | 支撑未来动作最关键的信息 |
| **跨实验信息（Cross-trial）** | 动态 | 积累经验（成功/失败），提供持久经验支撑 |
| **外部信息（External）** | 静态 | 缓解知识过时，拓展知识边界 |

### 记忆形式（Memory Forms）

两种呈现形式：文本形式 和 参数形式。

#### 文本记忆（Textual Form）

存储四类信息：
1. 完整的智能体-环境交互
2. 近期的智能体-环境交互
3. 检索出的智能体-环境交互
4. 外部知识

**研究重点**：
- **近期交互**：基于时间截断，强调 recency
- **检索记忆**：基于相关性、重要性、主题选择记忆内容
- **外部知识**：将数据库、百科、书本、知识库转化为自身记忆

> 注意：上下文不是无限长的，计算复杂度与长度呈二次增长，且长上下文中信息位置不同，被利用的程度也不同。

#### 参数记忆（Parametric Form）

不占上下文，主要有两种方法：

1. **微调（Fine-tuning）**
   - 优点：有监督微调，让智能体具有特定领域专家能力，不占用上下文，精度高、可靠性高
   - 缺点：容易过拟合，遗忘原来知识（灾难性遗忘），所需数据较大，只适用于离线

2. **记忆编辑（Knowledge Editing）**
   - 不是从特定数据集中提取领域知识
   - 针对仅需要调整的事实，不影响已存在的不相关知识
   - 适合小规模记忆调整，成本低，适合在线场景，可缓解灾难性遗忘

#### 文本记忆 vs 参数记忆对比

| 维度 | 文本记忆 | 参数记忆 |
|------|----------|----------|
| **有效性** | 内容全面，但受上下文长度限制；长上下文导致计算成本指数上升 | 不受上下文限制，但转换过程有信息损失，训练复杂 |
| **效率** | 写入效率高，读取效率低（增加提示词长度） | 写入效率低，读取效率高（隐式推理） |
| **可解释性** | 高 | 低 |
| **信息密度** | 低 | 高 |
| **适用场景** | 近期交互任务 | 大空间存储、长期熟悉的知识 |

**总结**：对于近期的交互任务，文本记忆更有效；对于需要大空间存储或长期、熟悉的知识，参数化记忆更有效。

### 记忆操作（Memory Operations）

记忆写入、记忆管理和记忆读取三者相互协同实现记忆功能。

#### 1. 记忆写入（Writing）

核心问题：选择哪些信息被记忆，如何表征。

- 原始信息容错冗余，包含大量噪声
- 不同环境反馈形式不同，如何提取并表征为记忆很重要

**代表作**：
- **SCM**：使用记忆控制器选择什么时机进行记忆写入
- **MemGPT**：由智能体自行控制记忆操作
- **MemoChat**：通过总结对话主题片段，将其作为索引记忆的键值

#### 2. 记忆管理（Management）

记忆通过反思生成高维度的记忆，合并冗余的记忆，遗忘不重要的记忆，从而实现管理。

![记忆操作研究工作统计](./images/table3-memory-operations.png)

#### 3. 记忆读取（Reading）

要进行下一步动作，需要提取相关的记忆信息，关键在于如何获取与当前状态相关的信息。

**代表作**：
- **ChatDB**：智能体生产一系列记忆链进行存储，在需要的时候通过 SQL 语句读取
- **MPC**：从记忆池中检索相关记忆，提供思维链，忽略特定记忆
- **Expel**：利用 FAISS 向量存储作为记忆池，获取相似度 Top-k 的记忆

> 记忆读取与记忆写入是协同的：文本记忆多采用文本相似度及其辅助信息读取；参数化记忆利用更新后的参数进行推理，是一种隐式推理。

## 如何评估记忆机制

两类评估方法：
1. **直接评估**：独立评测记忆模块的能力
2. **间接评估**：通过端到端的智能体任务进行评估

### 直接评估（Direct Evaluation）

#### 主观评价（Subjective Evaluation）

依靠人的主观判断衡量记忆模块的有效性，两个常见视角：
- **连贯性（Coherence）**：被召回的记忆是否与当前上下文匹配且自然
- **合理性（Plausibility）**：被召回的记忆内容是否合理

#### 客观评价（Objective Evaluation）

依据数值指标评价记忆模块的有效性，包含三个维度：

**1. 结果正确性（Result Correctness）**

$$
Correctness = \frac{1}{N}\sum_{i=1}^{N}\mathbb{I}\left[a_i=\hat{a}_i \right]
$$

其中匹配函数：

$$
\mathbb{I}\left[ a_i=\hat{a}_i \right] = \begin{cases}
  1 & \text{if } a_i=\hat{a}_i \\
  0 & \text{if } a_i \neq \hat{a}_i
\end{cases}
$$

**2. 引用准确性（Reference Accuracy）**

使用 F1 指标评估：

$$
\text{F1}=2 \cdot \frac{Precision \cdot Recall}{Precision + Recall}
$$

其中：
- $Precision=\frac{TP}{TP+FP}$
- $Recall=\frac{TP}{TP+FN}$

**3. 效率（时间和硬件成本）**

$$
\Delta{time}=\frac{1}{M}\sum_{i=1}^{M}t_i^{end}-t_i^{start}
$$

总时间成本包括记忆适配时间（写入+管理）和推理时间（读取时延）。

### 间接评估（Indirect Evaluation）

设计需要智能体高度依赖记忆的任务，常见任务：

1. **对话（Conversation）**
   - 评估连贯性（consistency）和参与度（engagement）
   - 参与度反映智能体回复的质量和吸引力，也反映智能体针对当前对话构建任务性格的能力

2. **多源问答（Multi-source QA）**
   - 评估来自任务内、跨任务以及外部知识的记忆信息
   - 检验智能体记忆在整合不同来源内容方面的能力

3. **长上下文应用（Long-context Applications）**
   - 段落检索是评估上下文能力的重要指标
   - 即在长上下文中找到给定问题或描述的重要段落
